Usecase Implementation: Edge Cases 🧪

1. The "Spider Web" (Multi-Actor & Inheritance)

Tests if markers point to the correct center-point of actors and handle overlapping lines. 

      usecaseDiagram
        actor "Guest" as G
        actor "Member" as M
        actor "Admin" as A
        
        %% Testing generalization markers (hollow arrows)
        generalization: M --> G
        generalization: A --> M

        system "Streaming Platform" {
          usecase "Browse Catalog" as UC1
          usecase "Play Video" as UC2
          usecase "Upload Content" as UC3
          usecase "Delete User" as UC4
        }

        G --> UC1
        M --> UC2
        A --> UC3
        A --> UC4
        
        %% Cross-linking
        M --> UC1

2. The "Logic Flow" (Include & Extend Inside System)

Checks if internal system links use the correct markers and curved pathing logic.

      usecaseDiagram
        actor "Customer" as C
        external "Stripe API" as Stripe

        system "Checkout System" {
          usecase "Place Order" as PO
          usecase "Apply Coupon" as AC
          usecase "Process Payment" as PP
          usecase "Authorize Card" as Auth
        }

        C --> PO
        include: PO --> PP
        extend: PO --> AC
        include: PP --> Auth
        dependency: Auth --> Stripe

3. The "Meta Data" (Notes, Anchors & Constraints)

Stress tests label wrapping and line-types for non-functional elements.

      usecaseDiagram
        actor "Security Auditor" as SA
        usecase "Audit Logs" as AL
        
        note "This process must be
encrypted at rest" as N1
        note "Check GDPR compliance" as N2

        SA --> AL
        
        %% Anchor (dashed line to element)
        anchor: N1 --> AL
        
        %% Constraint (dotted line between notes)
        constraint: N1 --> N2
